# Play Console Checklist — NutriFit V1 (Android-first)

Estado: operativo.
Última actualización: 2026-03-30.

## 1) Cuenta y organización

- [ ] Cuenta de desarrollador activa y en regla.
- [ ] App creada con nombre final y package name final.
- [ ] Usuario(s) con permisos mínimos (release manager, finance, support).

## 2) Store listing

- [ ] Short description y full description final.
- [ ] Ícono 512x512, feature graphic y screenshots actuales.
- [ ] Categoría y tags correctos (salud/bienestar, no claims médicos).
- [ ] URL de soporte configurada.
- [ ] URL de privacy policy configurada.

## 3) App content y cumplimiento

- [ ] Data Safety completado y consistente con implementación real.
- [ ] Content Rating completado.
- [ ] Target audience definido correctamente.
- [ ] Declaraciones de permisos (ej. micrófono) justificadas por funcionalidad.
- [ ] Política de no diagnóstico médico visible en metadata/app.

## 4) Monetización y pagos

- [ ] Payments profile creado y verificado.
- [ ] Suscripción `nutrifit_ai_monthly_499` activa.
- [ ] Base plan mensual activo con precio USD $4.99.
- [ ] Pruebas con testers licenciados (purchase, restore, cancel).
- [ ] Mensajería de precio/renovación/cancelación clara en paywall.

## 5) Testing tracks

- [ ] Internal testing listo para smoke de build.
- [ ] Closed testing creado (si aplica por políticas de cuenta nueva).
- [ ] Grupo de testers definido y con feedback loop.
- [ ] Criterios de promoción a producción documentados.

## 6) Release management

- [ ] AAB firmado subido (no debug signing).
- [ ] Notas de release claras.
- [ ] Mapeo de versión (`versionCode`/`versionName`) documentado.
- [ ] Pre-launch report revisado y issues críticos resueltos.
- [ ] Plan de rollback y hotfix definido.

## 7) Operación post-lanzamiento

- [ ] Dashboard de KPIs (installs, crash-free users, conversion premium, refunds).
- [ ] Canal de soporte operativo (SLA básico).
- [ ] Monitoreo de errores backend/billing activo.
